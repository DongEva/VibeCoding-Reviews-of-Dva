【2026.1.14】

一、AI工具使用体验（z.ai）

在这之前我是用过一个月的gemini。
我本人之前主要是TouchDesigner项目做的比较多，也有比较成熟的制作过小工具。所以我使用gemini时获得的体验感是相当好的。
它可以完好的实现代码零基础的我进行一些面部交互、体感交互游戏的制作，成为了一个非常非常好的技术辅助，我只需要在他基础上进行我的逻辑和美术的构建。确实也有感受到边界和细节丢失。

二、课程总结与实践感悟

课程里老师的总结说：
“指望让AI迅速马上的完成一个非常大的项目是不合理的，
很快就会碰到上限，
需要拆分流程，细化到各个环节。”
我深有同感，在课程开始之前我已经开始看datawhale社区的教程，也了解到建设更大的项目，其实应该引入spec coding框架，但是还没有深入了解。

在第一天写贪吃蛇小游戏的时候，我发现给他提一些更明确的需求。比如说红白游戏机这些，他其实对这些关键词的理解会更好。给他说精美这些词，其实是没有明确的效果的。
然后如果要新增弹窗这些东西，一定要明确弹窗的位置。

【2026.1.15】

一、金句分享

*"Live in the future, then build what's missing." （活在未来，然后构建缺失的东西。） —— Paul Graham，Y Combinator 联合创始人*

二、个人感悟

我觉得这句话非常有启发性，就把自己放在未来的情境里去看待现在的世界，有哪些东西是你会惊异的发现这些东西还没有变化，居然没有被优化的.那我觉得这些就是我们最亟待解决的，或者说最符合未来构想的一个方式。不管怎么样，他会有未来的思维，而不受现在的一些视角的局限.
走在路上我又想了想，这句话的这个角度其实是非常产品的角度，它不局限于网页小工具，而是说生活中的方方面面。非常的有开拓性。
